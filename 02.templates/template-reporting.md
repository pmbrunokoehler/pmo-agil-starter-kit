## Boas práticas

- Comece pela conclusão (o sumário), não pelos dados.
- Se o report nunca provoca uma decisão difícil, ele está escondendo problemas em vez de revelá-los.
- Produza primeiro a versão executiva (uma página). Derive versões mais detalhadas para outros públicos a partir dela.
- O report chega antes da reunião de governança, para que a reunião seja para decidir, não para atualizar.

> 📝 **[PREENCHER]:** ajuste as métricas-chave às que você de fato acompanha. As três sugeridas são pontos de partida comuns para PMO de portfólio.
